Tras este capítulo introductorio, el resto del documento se estructura como sigue: el
Capítulo 2 describe brevemente los conceptos necesarios, tales como qué es una \emph{Línea de Producto Software} o en qué consiste el \emph{Patrón Slicer} para poder entender la presente memoria. El Capítulo 3 explica la generación de código en la fase de \emph{Ingeniería del Dominio}. El Capítulo 4 describe el desarrollo de la generación de código para productos concretos en la fase de \emph{Ingeniería de Aplicación}. Por último, el Capítulo 5 analiza brevemente las conclusiones extraídas tras la realización
de este proyecto, así como posibles trabajos futuros.
