Tras este capítulo introductorio, el resto del documento se estructura como sigue. El Capítulo~\ref{chap:background} describe brevemente los conceptos necesarios para poder entender la presente memoria, y que no se pueden presuponer conocidos por el lector, tales como qué es una \emph{Línea de Producto Software} o en qué consiste el \emph{Slicer Pattern}. El Capítulo~\ref{chap:domain} explica el proceso de desarrollo de uno de los generadores de código creados, concretamente el que actúa durante la fase de la fase de \emph{Ingeniería del Dominio} del desarrollo de una línea de productos software. El Capítulo~\ref{chap:application} describe el desarrollo del generador de código que actúan durante la fase de configuración de una línea de productos software, la \emph{Ingeniería de Aplicaciones}. Dicho capítulo también comenta brevemente las acciones realizadas para el despliegue de la aplicación. Por último, el Capítulo~\ref{} sirve de sumario y cierre a esta memoria de Proyecto Fin de Carrera, proporcionando también las conclusiones extraídas tras su realización, así como posibles trabajos futuros.
