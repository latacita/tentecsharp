\chapterheader{Ingeniería de Aplicación}{Ingeniería de Aplicación}
\label{chap:application}

Este capítulo se describe la fase de \emph{ingeniería de aplicación} (en inglés, \emph{Application Engineering}) para nuestra línea de productos software. Dentro de dicha fase, se detalla el proceso seguido para componer las características creadas en la fase de ingeniería del dominio (ver Capítulo \ref{chap:domain}). A continuación, se describe la infraestructura creada para poder componer características siguiendo este proceso de forma tan automática como sea posible. Por último, se describen las acciones llevadas a cabos para realizar el despliegue de nuestro producto.

\section{Sumario}

Este capítulo ha mostrado los distintos pasos necesarios para realizar la composición de características en la plataforma .NET a través del lenguaje C\# y sus clases parciales. El primero de los pasos ha consistido en encontrar una modo de encapsular de forma separada el código creado en las fases de ingeniería de aplicación y dominio, para permitir derivar un número largo de aplicaciones concretas. Para tal cometido ha sido necesario desarrollar un mecanismo que supliese las carencias que existen en la plataforma .NET. En segunda instancia se ha desarrollado un metamodelo que permita crear modelos de un hogar inteligente, para que los usuarios pueden realizar configuraciones abstrayéndose de la implementación que subyace. Dicho metamodelo evita además que se creen configuraciones que no sean válidas. A continuación, para completar la automatización de nuestra línea de productos software, se presentan las plantillas de generación de código las cuales producen de forma automática el código necesario para componer diferentes características, de acuerdo con las necesidades de cada usuario. Por último, se expone el proceso seguido para desplegar esta línea de productos software, así como la página web creada como elemento de promoción de este proyecto.
