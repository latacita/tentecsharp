\chaptertoc

\section{Conclusiones}

Este proyecto ha desarrollado una línea de productos software para hogares automáticos y/o inteligentes mediante la tecnología .NET en el lenguaje C\#.
Se ha utilizado como mecanismo para soportar la variabilidad existente en una línea de productos las clases parciales de C\#. El principal objetivo de una línea de productos software es reducir los costes de desarrollo y mejorar la calidad de los productos software concretos pertenecientes a una familia de productos software.

No obstante, el presente proyecto no tenía como principal objetivo desarrollar una línea de productos software real. En cierto modo, se ha utilizado un caso de estudio como es el de la automatización de los hogares, debido a la gran variabilidad que dicho dominio posee. Por lo que dicho dominio es un excelente elemento de prueba para analizar debilidades y fortalezas de las clases parciales de C\# como elemento central para la construcción de líneas de productos software.

Durante el desarrollo del proyecto se han encontrado diversos problemas relacionados con el uso de las clases parciales como mecanismo para la modularización y composición de las distintas características de una línea de productos. Para suplir tales deficiencias, se han desarrollado estrategias alternativas que permitiesen solventar estos problemas. El principal objetivo de este proyecto ha sido demostrar que utilizando estas estrategias se pueden construir líneas de productos software bajo la tecnología .NET usando las clases parciales de C\# con ciertas propiedades ventajosas sobre una solución clásica basada en técnicas puramente orientadas a objetos.

El presente proyecto también demuestra que es posible automatizar una línea de productos software construida para la plataforma .NET, siguiendo un enfoque dirigido por modelos y las \emph{DSL Tools} proporcionadas por Microsoft. No obstante, el proceso de familiarización y aprendizaje relacionado con dichas herramientas, las \emph{DSL Tools}, puede ser bastante largo y costoso, debido principalmente, a la escasez, tanto en cantidad como en calidad, de la información proporcionada sobre su utilización.

Resumiendo, el presente proyecto puede considerarse como un referente para la creación de líneas de productos software bajo la tecnología .NET. De tal modo, que se han desarrollado métodos que permiten suplir las carencias que se encuentran al utilizar esta tecnología para la implementación de las líneas de productos software. Esto permite a las empresas actuales que trabajen con la plataforma .NET adquirir prácticas y hábitos relacionados con las líneas de productos software, sin necesidad de adquirir nuevas herramientas, entornos de desarrollo y sin necesidad, sobre todo, de aprender nuevos paradigmas de programación.

\section{Trabajos Futuros}

Todo proyecto es siempre susceptible de mejorar, existiendo una serie de características, que bien por falta de tiempo, por su excesiva complejidad o por ser de un interés secundario, nunca fueron incorporadas al proyecto o quedaron por resolver. Y como este proyecto no es ninguna excepción, a continuación se muestra una lista de trabajos futuros que se podrían realizar:

\begin{enumerate}
\item Como ya se comentó en el Capítulo~\ref{chap:domain}, el presente proyecto no trabaja con dispositivos domóticos reales. Se trata de una simple simulación de un hogar inteligente. Por lo que una posible línea de trabajo futuro consistiría en adaptar todos los elementos necesarios para que el presente proyecto funcionase sobre un hogar automatizado real, usando dispositivos reales.

\item Ampliar el número de características implementadas, intentado llegar a completar una línea de productos software para hogares inteligentes más amplia, donde aparezcan características como control inteligente de puertas, detección de incendios, acceso al \emph{Gateway} a través de dispositivos móviles o internet, entre otras muchas.

\item El proceso seguido para desarrollar este proyecto se puede aplicar a otros productos susceptibles de ser gestionados como una línea de productos software.
		
\end{enumerate}
